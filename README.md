# Class Attendance QR Check-in System for Lab Room 🎓⚡

ระบบเช็คชื่อเข้าห้องเรียนด้วย **QR Code** และ **Session Code** สำหรับห้องปฏิบัติการ (Lab Room) ออกแบบอย่างประณีตและพรีเมียมด้วยสี **Deep Blue (`#2C5EAD`)**, **Sky Blue (`#4BB8FA`)** และ **Soft Blue (`#C4E2F5`)** พัฒนาขึ้นในรูปแบบ **Single Page Application (SPA)** ด้วย HTML, Custom CSS และ JavaScript ES6 Modules โดยจัดเก็บข้อมูลไว้บน **LocalStorage** 100% ทำให้สามารถใช้งานแบบออฟไลน์หรือเปิดใช้งานบนคอมพิวเตอร์และมือถือได้ทันทีโดยไม่ต้องตั้งค่าเซิร์ฟเวอร์!

## ✨ ฟีเจอร์เด่นของระบบ (Key Features)

### 1. ระบบจัดการข้อมูลพื้นฐาน (CRUD Management)
* **จัดการรายวิชา:** อาจารย์สามารถ เพิ่ม แก้ไข ลบ และดูรายวิชาเรียนทั้งหมดได้ โดยข้อมูลจะแสดงผลเป็นแบบ Premium Cards
* **จัดการรายชื่อนักศึกษา:** สามารถกรองรายชื่อแยกตามวิชาเรียน เพิ่มนักศึกษาเป็นรายคน หรือลบนักศึกษาได้
* **ระบบนำเข้าแบบเร่งด่วน (CSV Import):** อาจารย์สามารถก๊อปปี้รายชื่อนักศึกษาจากไฟล์ Excel/CSV มาวางเพื่อนำเข้าข้อมูลนักศึกษาจำนวนมากในครั้งเดียวได้ทันที!

### 2. ระบบเช็คชื่อเข้าเรียน (QR Check-in & Live Monitor)
* **การเช็คชื่อแบบ Dynamic:** ระบบจะสร้างรหัสผ่าน **Session Code** สุ่มแบบไม่ซ้ำกัน และสร้าง **QR Code** ที่ฝังลิงก์นำทางอัตโนมัติ ให้นักศึกษาสแกนหรือกรอกเพื่อยืนยันตัวตน
* **หน้าจอจอโปรเจกเตอร์สำหรับห้องเรียน (Live Monitor Panel):** อาจารย์สามารถเปิดหน้าจอแสดง QR Code ขนาดใหญ่บนจอห้องเรียนได้ โดยมีตัวจับเวลาถอยหลัง (Extendable Countdown Timer) และกราฟอัตราส่วนผู้เช็คชื่อแบบ Real-time
* **Live Scanned Feed:** เมื่อนักศึกษาสแกนเช็คชื่อผ่านมือถือสำเร็จ รายชื่อจะเด้งกะพริบเป็นสีเขียวขึ้นบนจอของอาจารย์ทันทีโดยไม่ต้องกดรีเฟรชหน้าจอ!

### 3. ระบบยืนยันตัวตนและคำนวณสถานะอัตโนมัติ (Student View)
* **ข้อมูลที่ใช้ตรวจสอบตัวตน:** ใช้เพียง **รหัสนักศึกษา (Student ID)** เท่านั้น เมื่อระบบพบข้อมูลในฐานข้อมูล ระบบจะทำการ:
  * **ดึงชื่อ-นามสกุลอัตโนมัติ** แสดงผลการตรวจสอบตัวตนทันที
  * **ดึงหมู่เรียน/วิชาที่ลงทะเบียนอัตโนมัติ** เพื่อยืนยันว่านักศึกษาลงทะเบียนในรายวิชานั้นๆ ตรงกับคาบเรียน
  * **บันทึกการเช็คชื่อได้ทันที** โดยปุ่มยืนยันจะเปิดให้กดบันทึกได้หลังจากระบบยืนยันตัวตนสำเร็จ
* **คำนวณสถานะอัตโนมัติ:** เมื่อนักศึกษาเช็คชื่อ ระบบจะตรวจสอบเงื่อนไขเวลา:
  * **มาเรียน (Present):** เช็คชื่อก่อนเวลาเริ่มเรียนบวกเกณฑ์การเข้าสาย (Late threshold)
  * **สาย (Late):** เช็คชื่อเกินเวลาเปิดที่กำหนด
  * **ขาดเรียน (Absent):** นักศึกษาไม่มีบันทึกการสแกนในรอบเรียนนั้นๆ โดยระบบจะตั้งเป็นสถานะเริ่มต้นให้ทุกคนทันทีเมื่อสร้างรอบเรียน
* **ระบบป้องกันทุจริต:** ป้องกันนักศึกษาคนเดิมสแกนเช็คชื่อซ้ำซ้อนในรอบเดียวกัน และตรวจสอบว่ารหัสนักศึกษานั้นได้ลงทะเบียนเรียนในรายวิชานี้จริงหรือไม่

### 4. แดชบอร์ดและรายงานสรุป (Analytics & Reports)
* **Interactive Graphs:** แสดงสัดส่วนการมาสาย ขาด ลา และแนวโน้มอัตราการเข้าห้องเรียนย้อนหลัง 6 คาบล่าสุดด้วย **Chart.js**
* **Manual Override & Remarks:** อาจารย์สามารถแก้ไขสถานะของนักศึกษาด้วยตนเองได้ภายหลัง (เช่น เปลี่ยน ขาด เป็น ลา หรือเปลี่ยนสถานะหากทำเรื่องลาเรียน) พร้อมช่องสำหรับพิมพ์ข้อความหมายเหตุสั้นๆ
* **Excel-friendly Export (CSV Export):** ดาวน์โหลดตารางเช็คชื่อทั้งหมดออกมาเป็นไฟล์ `.csv` โดยระบบใส่ UTF-8 BOM (`\uFEFF`) ช่วยให้สามารถเปิดดูผ่าน **Microsoft Excel** ภาษาไทยได้ทันที ตัวอักษรไม่เพี้ยน!

## 🚀 วิธีการเข้าใช้งาน (Quick Start Instructions)

เนื่องจากระบบถูกเขียนขึ้นในรูปแบบ Pure Vanilla HTML, CSS และ Javascript:
1. **ดาวน์โหลดหรือเปิดโฟลเดอร์** `d:\Class Attendance QR Check-in System` บนคอมพิวเตอร์ของคุณ
2. **ดับเบิ้ลคลิกไฟล์** `index.html` เพื่อรันการทำงานผ่านเว็บเบราว์เซอร์ (Chrome, Firefox, Safari หรือ Edge) ทันที
3. **ระบบจะโหลดข้อมูลตัวอย่างสำเร็จรูป (Seed Data)** เข้าสู่ระบบให้ทันที เพื่อให้คุณทดสอบฟีเจอร์ กราฟวิเคราะห์ และรายงานได้ทันทีโดยไม่ต้องกรอกข้อมูลใหม่ทั้งหมดตั้งแต่ต้น!
4. **หากต้องการอัปโหลดขึ้นเว็บใช้งานจริง:** สามารถลากวางโฟลเดอร์นี้อัปโหลดขึ้น **Netlify** หรือ **Vercel** ได้โดยตรงโดยไม่ต้องมีขั้นตอนการ Build หรือ Complie ใดๆ ทั้งสิ้น

## 🛠️ แผนและรายละเอียดการพัฒนาระบบ (System Implementation Plans)

### 1. ระบบกล้องสแกน QR Code จริง & บันทึกพิกัด GPS (Camera QR Scanner & GPS Location Tracking)

พัฒนาระบบสแกนชื่อเข้าเรียนด้วยกล้องจริงและบันทึกพิกัด GPS สำหรับใช้บนมือถือเพื่อป้องกันการทุจริตในการเช็คชื่อ

* **Camera QR Scanner (การสแกนผ่านกล้องจริง)**: ใช้ไลบรารี `html5-qrcode` ในการเปิดใช้กล้องหลังของมือถือเพื่อสแกน QR Code และทำการสกัดแยก Session Code ออกมาเพื่อกรอกข้อมูลลงฟอร์มโดยอัตโนมัติ โดยมีแถบสวิตช์เลือกได้ระหว่างการเปิดใช้กล้องจริงและเครื่องจำลอง
* **GPS Location Capture (การดึงพิกัด GPS)**: ใช้ Geolocation API ในการตรวจจับค่าพิกัดละติจูดและลองจิจูดที่มีความแม่นยำสูงเมื่อนักศึกษากดส่งข้อมูลเช็คชื่อ และบันทึกเข้าไปในฐานข้อมูล LocalStorage
* **Visual & Maps Logging (การแสดงผลพิกัด)**: เพิ่มคอลัมน์พิกัด GPS ในหน้ารายงานอาจารย์ พร้อมเชื่อมโยงกับลิงก์แผนที่ Google Maps เพื่อเช็คตำแหน่งที่ตั้ง และรวมพิกัดในรายงานการส่งออกไฟล์ CSV

### 2. ระบบกำหนดความจุผู้เรียนในรายวิชา (Course Student Capacity Limits)

กำหนดขนาดความจุของนักเรียนที่รับได้ในรายวิชาเพื่อจำกัดโควตาการลงทะเบียนเรียนและจำลองข้อจำกัดจริงในห้องปฏิบัติการ

* **Database Capacity Check**: บันทึกฟิลด์ `max_students` ไว้ในแต่ละวิชา (ค่าเริ่มต้น 40 คน) และทำการตรวจสอบจำนวนคนลงทะเบียนล่าสุด หากมีจำนวนถึงขีดจำกัดสูงสุด ระบบจะทำการบล็อกปุ่มบันทึกและแสดงกล่องแจ้งเตือนความผิดพลาดทั้งในฟอร์มเดี่ยวและฟอร์มนำเข้า CSV
* **Roster Ratio Subtitles**: แสดงสัดส่วนนักเรียนที่ลงทะเบียนเรียนจริงต่อความจุสูงสุด (เช่น `8 / 40 คน (ความจุ)`) บนการ์ดวิชาเรียนและในหน้าจัดการนักเรียน
